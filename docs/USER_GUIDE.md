# NaviDash 用户指南

欢迎使用 NaviDash。本文档基于当前版本的实际产品形态整理，帮助你快速了解页面结构、常见操作和数据管理方式。

## 界面结构

NaviDash 当前主要由三部分组成：

1. 顶部栏
   包含搜索框、组件库开关、编辑模式开关和全局设置入口。
2. 左侧组件库
   用于搜索、浏览和添加可用 widget。
3. 主画布
   用于摆放和展示 widget，也是你日常使用的首页区域。

## 基本使用

### 搜索

- 顶部搜索框支持直接输入关键词并跳转到所选搜索引擎
- 可以切换不同搜索引擎
- 支持快捷键 `Ctrl + K` 聚焦搜索框

### 进入编辑模式

- 点击顶部栏中的“自定义 / 完成”按钮切换编辑模式
- 编辑模式下可以移动 widget、打开 widget 设置、删除 widget

### 打开组件库

- 点击顶部栏左侧的“组件库”按钮
- 组件库中可以搜索 widget，也可以直接点击添加
- 某些 widget 添加后会自动打开设置面板，方便首次配置

## Widget 操作

### 添加 widget

1. 打开组件库
2. 找到想添加的 widget
3. 点击添加，或在支持的交互中拖拽到主画布

当前内置 widget 包括：

- `Clock`
- `Weather`
- `Date`
- `Calendar`
- `Todo`
- `Memo`
- `Quick Link`
- `Links`
- `Photo Frame`

### 移动 widget

1. 进入编辑模式
2. 按住 widget 左上角的拖拽手柄
3. 拖到目标位置后松开

当目标位置已有其他 widget 时，系统会自动进行重排和避让。

### 调整 widget 内容

不同 widget 的编辑方式不同：

- `Memo` 可以直接在组件内编辑内容，并自动保存
- `Todo` 可以直接在组件内添加、勾选和删除任务
- `Clock`、`Weather`、`Links`、`Quick Link`、`Photo Frame` 等通过设置面板修改配置

### 打开 widget 设置

1. 进入编辑模式
2. 点击 widget 右上角的设置按钮
3. 在弹出的设置面板中修改尺寸或配置

## 常用 widget 说明

### Clock

- 用于显示当前时间
- 当前提供多种样式配置
- 不同尺寸下展示密度会不同

### Weather

- 通过服务端 `/api/weather` 代理获取天气信息
- 推荐通过环境变量统一配置天气服务参数
- widget 内主要填写城市或经纬度

如果天气没有显示，请优先检查：

- `.env` 中是否已正确填写 `QWEATHER_API_KEY`
- 容器或开发服务是否已重启
- 城市或经纬度是否正确

### Memo

- 支持直接输入内容
- 支持轻量 Markdown 样式内容，例如标题、列表、链接、引用
- 组件会自动保存内容

### Todo

- 可在组件内直接添加待办项
- 可勾选完成
- 可删除单项，也可清除已完成项

### Links / Quick Link

- `Quick Link` 适合单个常用入口
- `Links` 适合一组常用站点或服务入口

### Photo Frame

- 可配置一张或多张图片
- 支持自动轮播和间隔设置

## 全局设置

点击顶部栏中的“设置”按钮可以打开全局设置面板。

当前主要分为三类：

- 外观
- 语言
- 数据工具

### 外观

可调整内容包括：

- 背景预设
- 自定义背景图
- 模糊度
- 遮罩透明度
- 主题色
- 页面标题
- Favicon

### 语言

- 可切换当前界面语言

### 数据工具

支持以下能力：

- 导出当前配置为 JSON
- 从 JSON 导入配置
- 重置本地数据

导入会覆盖当前配置，建议在导入前先导出备份。

## 数据保存

NaviDash 目前以本地部署为主，运行数据通常保存在宿主机挂载目录中。

主要包括：

- 全局设置
- widget 布局
- widget 配置

如果你使用 Docker，建议把数据目录挂载到仓库外，例如 `/opt/navidash-data`。

## Demo 模式

如果启用了 Demo 模式：

- 页面可正常浏览和试用
- 前端交互仍可进行
- 刷新后会恢复为默认演示内容
- 所有写入不会真正持久化

这更适合在线展示，不适合作为长期使用环境。

## 常见问题

### 为什么刷新后我的修改消失了？

通常有两种可能：

- 当前运行在 Demo 模式
- 持久化目录没有正确挂载或没有写权限

### 为什么天气组件没有数据？

请检查：

- `QWEATHER_API_KEY` 是否已配置
- 容器是否已重启
- 城市名或经纬度是否有效
- 网络是否可以访问天气服务

### 如何完整备份当前首页？

打开设置面板，在“数据工具”中导出 JSON。  
如果你是 Docker 部署，也建议同时备份宿主机上的数据目录。

### 如何恢复到干净状态？

可以在设置面板中使用重置功能。  
如果需要彻底恢复，也可以删除挂载目录中的运行数据后重新启动容器。
